# MEMORY.md

코드나 git 히스토리만으로는 알 수 없는 프로젝트 맥락(결정, 이유, 진행 상황)을
기록하는 문서입니다. 새 세션(팀원 또는 Claude Code)이 이 파일을 읽으면
"왜 지금 이 상태인지"를 빠르게 파악할 수 있어야 합니다.

작성 규칙:
- 시간순이 아니라 주제별로 정리합니다. 오래된/무효화된 항목은 삭제하거나 수정합니다.
- "무엇을 했다"가 아니라 "왜 그렇게 했다/하기로 했다"를 남깁니다.
- 코드를 읽으면 알 수 있는 내용(구조, 파일 위치)은 적지 않습니다 — `CLAUDE.md`, `.claude/rules/` 참고.

## 결정 사항

- `games`/`game-stats`/`standings` 목록 조회 API는 배열이 아니라 `{ data, total, page, limit, totalPages }` 형태의 `PaginatedResult`를 반환한다. 응답 크기가 무제한이라 컨슈머가 한 번에 전체를 받아야 했던 문제를 해결하기 위함.
